 ## Usage Instructions
 
 ### Referencing during Phase 3 Recommendation
 When recommending a design direction, you can showcase prefabricated screenshots corresponding to the scenario:
 ```
 "Here is how a Newsletter Cover would look under the Pentagram style → [Show cover/cover-pentagram.png]"
 "A PPT Data Slide in the Takram style feels like this → [Show ppt/ppt-takram.png]"
 ```
 
 ### Scenario Allocation Priority
 1. Precision match for User Request → Directly show corresponding scenario.
 2. Partial match (similar type) → Show the closest scenario (e.g. "Product site" → show SaaS Landing Page).
 3. Zero match → Skip prefabs, move directly to Phase 3.5 on-the-ground generative mockups.
 
 ### Horizontal Comparison
 It is highly effective to show all 3 styles of a scenario side-by-side to help the user directly compare:
 - "Here is the same Newsletter Cover rendered in 3 distinct philosophies."
 - Viewing order: Pentagram (Rational Restraint) → Build (Luxury Minimalism) → Takram (Soft & Warm).
